### Vulnerability List

- Vulnerability Name: LaTeX Injection via Identifier Names
- Description:
    1. An attacker crafts a Python function definition with a specially crafted identifier name.
    2. This identifier name contains malicious LaTeX commands, such as `\documentclass{article}\usepackage{osystem}\system\pause\input{|"bash -c 'touch /tmp/pwned'"}`, disguised as a seemingly normal identifier.
    3. The attacker uses `latexify` to convert this Python function into LaTeX.
    4. The `IdentifierConverter` in `latexify` converts the identifier name into LaTeX, but it does not sanitize or escape LaTeX special characters within the identifier name itself, if `escape_underscores=False`.
    5. When the generated LaTeX code is compiled (e.g., using pdflatex), the malicious LaTeX commands within the identifier name are executed. In this example, it would execute the bash command `touch /tmp/pwned` on the server compiling the LaTeX.
- Impact:
    - **High**: Arbitrary command execution on the server or system compiling the generated LaTeX document. This can lead to complete system compromise, data exfiltration, or denial of service.
- Vulnerability Rank: Critical
- Currently Implemented Mitigations:
    - The project uses `escape_underscores=True` by default in `Config.defaults()` (in `/code/src/latexify/config.py`). When `escape_underscores=True`, underscores in identifiers are escaped, which might prevent some basic injection attempts that rely on underscores for subscripts, but it does not prevent injection through other LaTeX commands in identifiers when `escape_underscores=False`.
- Missing Mitigations:
    - **Input Sanitization**: The library lacks proper sanitization of identifier names to prevent LaTeX injection. It should escape or reject identifier names containing LaTeX special characters or commands, regardless of the `escape_underscores` setting.
    - **Output Encoding**: While underscores are escaped when `escape_underscores=True`, this is insufficient. All LaTeX special characters and commands within identifiers should be systematically escaped or removed to ensure safe LaTeX output, or identifiers should be validated against a strict whitelist.
- Preconditions:
    - The attacker can control the Python code snippet that is processed by `latexify`.
    - The `escape_underscores` parameter is set to `False` either directly by the user or through a configuration that disables underscore escaping. While default is `True`, users can change it via parameters.
    - The generated LaTeX output is compiled using a LaTeX compiler (pdflatex, lualatex, etc.).
- Source Code Analysis:
    1. **`/src/latexify/config.py`**: Defines the `Config` class with `escape_underscores` parameter, defaulting to `True`.
    2. **`/src/latexify/generate_latex.py`**: The `get_latex` function uses the `Config` to control LaTeX generation.
    3. **`/src/latexify/codegen/identifier_converter.py`**: The `IdentifierConverter.convert` method (lines 63-92) handles the conversion of Python identifiers to LaTeX. When `escape_underscores=False`, it does not perform any sanitization of the identifier name itself beyond handling underscores.
    ```python
    def convert(self, name: str) -> tuple[str, bool]:
        ...
        if not self._escape_underscores and "_" in name:
            # Check if we are going to generate an invalid Latex string. Better to
            # raise an exception here than have the resulting Latex fail to
            # compile/display
            name_splits = name.split("_")
            if not all(name_splits):
                raise ValueError(
                    "Neither preceding/trailing underscores nor double underscores is "
                    f"allowed by the `escape_underscores` option, but got: {name}"
                )
            elems = [
                IdentifierConverter(
                    use_math_symbols=self._use_math_symbols,
                    use_mathrm=False,
                    escape_underscores=True,
                ).convert(n)[0]
                for n in name_splits
            ]
            # Wrap sub identifiers in nested braces
            name = "_{".join(elems) + "}" * (len(elems) - 1)

        if self._use_math_symbols and name in expression_rules.MATH_SYMBOLS:
            return "\\" + name, True

        if len(name) == 1 and name != "_":
            return name, True

        escaped = name.replace("_", r"\_") if self._escape_underscores else name # Only escapes underscores
        wrapped = rf"\mathrm{{{escaped}}}" if self._use_mathrm else escaped # Wraps in \mathrm if needed

        return wrapped, False
    ```
    As seen in the code, if `escape_underscores` is `False`, only underscore escaping is skipped. No other sanitization is performed on the identifier name, allowing LaTeX injection if the identifier itself contains malicious commands.
    4. **`/src/latexify/codegen/function_codegen.py` and `/src/latexify/codegen/expression_codegen.py`**: These codegen classes use `IdentifierConverter` to convert identifiers, passing user-controlled identifier names directly to the LaTeX output without further checks.
