### 1. Vulnerability Name: Stack Overflow in Recursive Parsing of Nested Expressions

- Description:
    - The `rfmt` application uses PLY (Python Lex-Yacc) to parse R code. The grammar defined in `/code/inst/python/rfmt/r_language.py` for parsing R expressions includes recursive rules, particularly in the `expr` non-terminal.
    - An attacker can craft a deeply nested R expression, such as deeply nested parentheses or function calls, that when parsed by `rfmt` leads to excessive recursion in the parsing process.
    - This excessive recursion can exhaust the call stack, resulting in a stack overflow error and causing the `rfmt` application to crash.
    - Steps to trigger vulnerability:
        1. Create a malicious R file containing a deeply nested expression.
